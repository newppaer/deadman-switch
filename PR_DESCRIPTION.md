# DeadManSwitch 功能增强与界面优化

## 概述
本次更新全面增强了 DeadManSwitch 应用的功能性和用户体验，包括活动检测改进、界面美化、性能优化等。

## 主要变更

### 1. 增强活动检测
- **添加 UsageStatsManager 支持**：除了屏幕解锁，现在还能检测应用使用情况
- **多维度检测**：结合屏幕解锁、应用使用等多重信号
- **智能检测逻辑**：5分钟内检测到任意活动即视为活跃

### 2. 界面全面升级
- **Material3 设计语言**：采用最新 Material Design 3 组件
- **深色模式支持**：自动跟随系统或手动切换
- **底部导航栏**：状态、联系人、设置三标签页
- **现代化卡片设计**：圆角、阴影、渐变背景
- **响应式布局**：适配不同屏幕尺寸

### 3. 通知系统重构
- **多通道通知**：分离监控通知和警报通知
- **自定义振动模式**：紧急警报有独特振动
- **优先级管理**：警报通知为高优先级
- **持久化通知**：警报不会自动取消

### 4. 功能模块化
- **ActivityDetector**：独立的活动检测工具类
- **AppNotificationManager**：通知管理工具类
- **Contact 数据模型**：为紧急联系人功能准备

### 5. 用户体验优化
- **实时状态更新**：无活动时长每10秒刷新
- **直观状态指示**：颜色编码状态卡片
- **详细统计信息**：今日/本周/本月活动统计（占位）
- **设置页面**：开机自启动、深色模式开关

### 6. 国际化支持
- **字符串资源化**：所有文本提取到 strings.xml
- **中文本地化**：完整的中文界面

## 技术改进

### 依赖更新
- 添加 navigation-compose 用于底部导航
- 添加 room-runtime 为联系人功能准备
- 更新 Compose BOM 版本

### 代码结构优化
- 分离业务逻辑和 UI 层
- 使用 Kotlin Coroutines 处理异步
- 遵循 Compose 最佳实践

### 权限扩展
- `PACKAGE_USAGE_STATS`：应用使用统计
- `SEND_SMS`：未来短信通知
- `CALL_PHONE`：未来电话通知
- `ACCESS_NETWORK_STATE`：网络状态检测

## 构建说明
- **目标 API**: 34 (Android 14)
- **最低 API**: 26 (Android 8.0)
- **构建工具**: Gradle + Kotlin DSL
- **主要框架**: Jetpack Compose + Material3

## 测试建议
1. **活动检测**：解锁屏幕、打开应用、长时间无操作
2. **通知系统**：监控通知、警报通知、振动模式
3. **界面交互**：底部导航、设置切换、深色模式
4. **服务稳定性**：长时间运行、设备重启
